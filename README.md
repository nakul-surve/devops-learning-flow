 
# DevOps Learning Flow

This repository explains the end-to-end DevOps workflow in simple terms.

## Step 1: Code
- Application code is written and version controlled using Git.

## Step 2: Continuous Integration
- Code changes trigger automated checks using CI tools (GitHub Actions).

## Step 3: Build
- Docker builds container images for consistent environments.

## Step 4: Deploy
- Containers are deployed on cloud servers (AWS).

## Step 5: Run & Monitor
- Applications run on Linux servers and are monitored for issues.

## Tools Covered
- Git & GitHub
- GitHub Actions
- Docker
- AWS
- Linux

## Learning Goal
Understand the full DevOps lifecycle instead of isolated tools.
